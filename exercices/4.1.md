# Exercice 4.1 : Travailler en équipe au jour le jour

## Ressources

- [https://www.markdownguide.org/basic-syntax/](https://www.markdownguide.org/basic-syntax/) syntaxe de bases pour les fichiers markdown, pas obligatoire pour cet exercice.

## Enoncé

## Partie I : commit

1. A la suite de l'exercice 2.1, créez un fichier *README.md* contenant la liste des ingrédients de votre sandwich préféré, un ingrédient par ligne comme dans l'exemple ci-après.

```markdown
- Steak
- Salade
- Tomate
- Cornichon
- Fromage
```
2. Exécutez la commande `git status` pour voir l'état de votre dépôt.
3. Effectuez votre premier commit avec la commande `git commit -m "first commit"`
4. Ajoutez une image de votre sandwich préféré dans le dossier */img*
5. Regardez l'état de votre dépôt
6. Effectuez un second commit avec un message court et clair de vos modifications.
7. Ajoutez au moins 3 nouvelles photos, effectuez un commit par nouvelle photo.

---

## Partie II : tag

8. Utilisez la commande `git tag --help` pour voir la documentation de cette commande
9. Ajoutez un nouveau tag v1.0.0 associé au dernier commit
10. Affichez la liste des tags.
11. Créez un nouveau fichier burger.md avec la liste des ingrédients
12. Effectuez un commit et créer un nouveau tag v1.0.1
13. Créez un dépôt distant sur GitHub et laissez-vous guider par les commandes de GitHub pour faire le lien avec votre dépôt distant et votre dépôt local. 


---

## Partie III : push/pull

1. Envoyez vos fichiers vers le dépôt distant à l'aide de la commande `git push`
2. Effectuez les modifications en ligne du fichier *README.md* depuis GitHub en ajoutant dans le *README.md* sandwich végan
3. Récupérez les modifications effectuées en ligne en local à l'aide de la commande `git pull`
4. Ouvrez le *README.md* et vérifiez que vous avez bel et bien le contenu du sandwich végan

---

## Partie IV : blame et diff

1. Comparez l'état de votre dépôt entre le premier et le dernier commit à l'aide des hashs associées à chaque commit à l'aide de la commande `git diff [hash_premier_commit] [hash_dernier_commit]`
2. Affichez les changements de manière plus détaillée du fichier *README.md*
