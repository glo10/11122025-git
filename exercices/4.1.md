# Exercice 4.1 : Travailler en équipe au jour le jour

## Ressources

- [https://www.markdownguide.org/basic-syntax/](https://www.markdownguide.org/basic-syntax/) syntaxe de bases pour les fichiers markdown, pas obligatoire pour cet exercice.

## Enoncé

## Partie I : commit

1. A la suite de l'exercice 2.1, créez un fichier *README.md* contenant la liste des ingrédients de votre sandwich préféré, un ingrédient par ligne comme dans l'exemple ci-après.

```markdown
- Steak
- Salade
- Tomate
- Cornichon
- Fromage
```
2. Exécutez la commande `git status` pour voir l'état de votre dépôt.
3. Effectuez votre premier commit avec la commande `git commit -m "first commit"`
4. Ajoutez une image de votre sandwich préféré dans le dossier */img*
5. Regardez l'état de votre dépôt
6. Effectuez un second commit avec un message court et clair de vos modifications.
7. Ajoutez au moins 3 nouvelles photos, effectuez un commit par nouvelle photo.

---

## Partie II : tag

8. Utilisez la commande `git tag --help` pour voir la documentation de cette commande
9. Ajoutez un nouveau ***tag v1.1.0*** associé au dernier commit
10. Affichez la liste des tags.
11. Créez un nouveau fichier burger.md avec la liste des ingrédients
12. Effectuez un commit et créer un nouveau ***tag v1.1.0***
13. Créez un dépôt distant sur GitHub et laissez-vous guider par les commandes de GitHub pour faire le lien avec votre dépôt distant et votre dépôt local. 


---

## Partie III : push/pull

14. Envoyez vos fichiers vers le dépôt distant à l'aide de la commande `git push`
15. Effectuez les modifications en ligne du fichier *README.md* depuis GitHub en ajoutant dans le *README.md* un sandwich végan et effectuez un commit
16. Récupérez les modifications effectuées en ligne en local à l'aide de la commande `git pull`
17. Ouvrez le *README.md* et vérifiez que vous avez bel et bien le contenu du sandwich végan

---

## Partie IV : blame et diff

18. Comparez l'état de votre dépôt entre le premier et le dernier commit à l'aide des hashs associées à chaque commit à l'aide de la commande `git diff [hash_premier_commit] [hash_dernier_commit]`
19. Affichez les changements de manière plus détaillée du fichier *README.md*

### Aide

Pour rediriger le sortie des commandes précédentes vers un fichier plus facile à traiter, vous pouvez combiner votre commande git avec `> nom_fichier.txt`.
Par exemple `git blame README.md > blame_readme.txt`

---

## Partie V : git format-patch

20. Utilisez la commande `git format-patch HEAD -1 -o ./patches` pour extraire les changements du dernier commit dans un fichier *0001-[message-commit].patch* dans le dossier */patches/* (nouvellement crée)
21. Créez un nouveau dossier en dehors votre projet git encore et initialisez un nouveau projet git
- Attention à ne pas initialiser un dépôt git dans un autre dépôt git
22. Effectuez votre premier commit sur ce nouveau dépôt
23. Copiez/collez le dossier */patches/* du premier dépôt au second dépôt (à la racine du projet)
24. Depuis le nouveau dépôt local, exécutez la commande `git am --3way ./patches`
25. Observez le résultat et regardez l'état de votre dépôt ainsi que l'historique des commandes
